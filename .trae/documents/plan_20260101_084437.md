1. 分析问题：

   * 文件上传失败，错误信息："No file uploaded"

   * axios实例默认设置了Content-Type为application/json

   * 显式设置Content-Type为multipart/form-data可能导致边界问题

2. 修复方案：

   * 修改uploadSingleFile函数，完全移除Content-Type设置，让axios自动处理

   * 确保FormData正确构造

   * 检查并确保上传请求正确发送文件数据

3. 具体修改：

   * 在uploadSingleFile函数中，创建一个新的headers对象，只包含认证信息

   * 不添加任何Content-Type，让axios自动处理multipart/form-data

   * 确保FormData正确调用append方法

4. 预期结果：

   * axios自动添加正确的multipart/form-data Content-Type和边界

   * 服务器能够正确接收文件

   * 上传失败："No file uploaded"错误消失
