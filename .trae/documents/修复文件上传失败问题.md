1. 分析问题：上传失败是因为axios默认设置了'Content-Type': 'application/json'，但上传文件需要使用multipart/form-data格式

2. 修复方案：

   * 修改`uploadSingleFile`函数，确保在上传文件时不覆盖axios的自动Content-Type设置

   * 确保FormData正确发送文件数据

3. 具体修改：

   * 在`uploadSingleFile`函数中，检查并移除headers中的'Content-Type'，让axios自动处理multipart/form-data

   * 或者显式将Content-Type设置为multipart/form-data，但不指定边界

4. 预期结果：

   * 文件上传时使用正确的Content-Type

   * 服务器能够正确接收并处理文件

   * 上传失败："No file uploaded"错误消失
