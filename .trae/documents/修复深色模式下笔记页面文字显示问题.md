# 修复深色模式下笔记页面文字显示问题

## 问题分析
根据用户反馈，深色模式下笔记页面的主要文字显示为蓝色，导致视觉不可见。通过分析Notes.vue文件，我发现问题可能出在以下几个方面：

1. **玻璃面板样式**：.glass-panel类使用了固定的背景色和文字颜色，没有使用主题系统中的CSS变量
2. **v-md-editor样式覆盖不完整**：编辑区域的文本颜色可能没有正确适配深色模式
3. **硬编码颜色值**：部分元素使用了硬编码的蓝色值，没有随主题变化
4. **深色模式适配不完整**：@media查询可能没有覆盖所有文本颜色设置

## 修复方案

### 1. 更新玻璃面板样式
将.glass-panel类的样式从固定颜色值改为使用主题系统中的CSS变量，确保与当前主题保持一致。

### 2. 完善v-md-editor样式覆盖
确保v-md-editor在深色模式下的所有文本元素都使用正确的颜色值，包括：
- 编辑区域文本
- 预览区域文本
- 工具栏按钮和图标
- 代码块和语法高亮

### 3. 替换硬编码颜色值
将所有硬编码的颜色值（特别是蓝色）替换为主题系统中的CSS变量，确保随主题变化。

### 4. 统一深色模式适配
移除@media (prefers-color-scheme: dark)查询，改用主题系统的CSS变量，确保与全局主题保持一致。

### 5. 确保所有文本元素使用主题变量
检查并更新所有文本元素的颜色设置，确保使用--text-primary、--text-secondary等主题变量。

## 预期效果
- 深色模式下笔记页面的主要文字颜色正确，与背景有足够的对比度
- 所有元素的颜色随主题变化，保持一致的视觉效果
- 编辑区域和预览区域的文本颜色正确适配深色模式
- 与全局主题系统保持一致，支持主题切换

## 实施步骤
1. 修改Notes.vue中的.glass-panel样式，使用主题变量
2. 完善v-md-editor的样式覆盖，确保编辑区域文本颜色正确
3. 替换所有硬编码的颜色值为主题变量
4. 移除@media查询，改用主题系统的CSS变量
5. 测试主题切换功能，确保笔记页面正确响应
